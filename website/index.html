How it works

Three steps. Under 50ms.

01

Features

Everything you need. Nothing you don't.

10 rule types

Amount limits, vendor allowlists, time windows, geography blocks, discount caps, and duplicate detection. Mix and match.

Immutable audit log

Every decision is logged with 14 fields. Can never be edited or deleted. Filter by agent, action type, or date range.

Policy versioning

Every update creates a new version. Previous versions stay intact. You always know which rules were in effect.

Duplicate guard

Catches when an agent submits the same action twice. Configurable windows prevent retry loops from creating duplicates.

Semantic classification

When your agent says "make the customer whole," SpendGuard understands the intent and applies the right rules.

MCP ready

AI agents discover and use SpendGuard through the Model Context Protocol. Connect it to Claude and check policies naturally.

Security

Built for financial data

API key hashing

SHA-256 hashed on creation. The original key is shown once and never stored on our servers.

Encrypted in transit

Every API call is encrypted with TLS 1.2 or higher. Unencrypted connections are rejected.

Append-only audit

Once a decision is recorded, it can never be modified or deleted by anyone.

Injection protected

All database queries are parameterized. No user input ever touches raw SQL.

Rate limiting

Per-key and per-IP limits that persist across server restarts. Cannot be bypassed.

Zero PII storage

We only store action metadata. No bank accounts, no SSNs, no sensitive customer data.
